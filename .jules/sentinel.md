## 2025-05-02 - Missing Authentication on Admin Endpoints
**Vulnerability:** The Vercel serverless function endpoints `/api/admin` and parts of `/api/courses` utilized the Supabase Service Role Key (which circumvents Row-Level Security) without validating the caller's identity or authorization level. This allowed any unauthenticated user to send POST requests and manipulate the database (add tools, news, generate courses).
**Learning:** Even when the backend correctly initializes a powerful admin client, it is imperative to implement a layer of authorization checking for any protected route. Using the service role key doesn't inherently secure the endpoint; it just gives the server script elevated privileges.
**Prevention:** Always verify incoming JWTs (via `supabase.auth.getUser(token)`) and implement custom role-checking (e.g., querying `user_profiles` for an `admin` role) prior to performing sensitive database modifications. Add authorization headers securely from the frontend for operations requiring admin privileges.
